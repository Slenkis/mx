Система безопасности Windows Server


№ Вопрос Ответ
1 Прежде чем связаться с сервером для подтверждения своей личности пользователь от службы Kerberos — центра распространения ключей (Key Distribution Center, KDC) — запрашивает билет

9 При наличии протокола аутентификации Kerberos служба Kerberos на сервере проверяет подлинность пользователя
10 В Windows 2003 протокол Kerberos является службой аутентификации по умолчанию и основным протоколом безопасности
11 Шифрованная файловая система позволяет пользователям шифровать и расшифровывать файлы и применяется для защиты файлов от злоумышленников.

15 Технология Authenticode – это элемент безопасности в Microsoft Internet Explorer, гарантирует аутентификацию программных компонентов в Интернете
16 Data Encryption Standard-Cipher Chaining (DES-CBC) – это алгоритм шифрования с применением закрытого ключа. генерируемое случайное число используется совместно с закрытым ключом для шифрования информации
17 Шифрование и расшифровка — операции уязвимые. Поэтому EFS выполняет все операции автоматически
18 В соответствии с терминологией Kerberos, сеансовый ключ — это временный ключ шифрования используется двумя участниками безопасности ограниченное время в течение единственного сетевого соединения
19 Серверное ядро службы сертификации – это главный компонент службы сертификации, выступающий как посредник для всех запросов, получаемых от входных модулей, направляя потоки информации между компонентами во время обработки запроса и генерации сертификата
20 Сертификат, выданный центром сертификации самому себе, называется корневым
21 Секретный ключ обычно используется в особых сеансах связи или на короткое время, после чего аннулируется
22 Криптография – это набор математических методов для шифрования и расшифровки данных
23 Для установки сертификата на клиентской системе используется Web-страница http://имя_cepвepa/certsrv
24 Открытый и закрытый ключи обычно применяются для шифрования данных и цифровой подписи сообщения
25 EFS использует комбинацию открытого и закрытого ключей пользователя, а также выбранный случайным образом ключ шифрования файла (file encryption key, FEK)
26 Безопасность системы сертификатов зависит от защищенности закрытых ключей
27 Существуют следующие типы центров сертификации корневой ЦС предприятия; подчиненный ЦС предприятия; изолированный корневой ЦС; изолированный подчиненный ЦС

30 Hash Message Authentication Code (HMAC) – это алгоритм шифрования с применением закрытого ключа, обеспечивающий целостность, аутентификацию и предотвращение повторов
31 Rivest, Shamir. Adleman(RSA) – это базовый алгоритм шифрования, поддерживающий цифровые подписи, распределенную аутентификацию, прием закрытого ключа с помощью открытого, шифрование больших объемов данных без предварительной передачи секретной информации

При этом каждый объект будет иметь только один (собственный) закрытый ключ, но сможет получать множество открытых ключей, парных к другим закрытым ключам

37 Digital Signature Standard (DSA) - это алгоритм шифрования с применением открытого ключа для генерации цифровых подписей
38 При отправлении сообщения создается его выборка — представление сообщения вроде контрольной суммы (cyclic redundancy check, CRC). Для зашифровки выборки сообщения отправитель применяет свой закрытый ключ
39 В соответствии с терминологией Kerberos, билет (ticket) — это запись, позволяющая клиенту аутентифицировать себя для сервера
40 Смарт-карты применяются для хранения открытого и закрытого ключа пользователя и сертификата
41 Выпускаемый службой Kerberos билет включает: сеансовый ключ; имя пользователя, для которого выдан сеансовый ключ; срок действия билета
42 Протоколы ISAKMP/Oakley (IKE) проводят согласование и устанавливают между компьютерами сопоставление безопасности связи
43 В соответствии с терминологией Kerberos, участник безопасности (principal) — это имеющий уникальное имя пользователь, клиент или сервер, участвующий в сеансе связи
44 Аутентификация – это процесс надежного определения подлинности подключающегося компьютера или пользователя
45 Синтаксис и формат сертификата определяет стандарт Х.509

47 Для конфигурации системы безопасности применяются оснастки Security Configuration And Analysis, Security Templates и Group Policy
48 База данных сервера службы сертификации – это БД, которая хранит информацию о текущем состоянии и журнал со всеми выпущенными сертификатами и списками отзыва сертификатов (CRL)
49 Целостность подразумевает сохранение информации в том виде, в котором она была передана
50 Аудит в Windows — это процесс отслеживания действий пользователя и действий Windows
51 Цифровая подпись обеспечивает аутентификацию и проверку целостности данных
52 Шифрование с применением открытых ключей представляет собой асимметричную схему шифрования с парой ключей
53 Фильтры IP руководят действиями в зависимости от пункта назначения пакета IP, активного IP-протокола и используемых портов
